# ZZZ API

Unofficial Zenless Zone Zero catalog API for fan apps.

This project provides static game catalog data, Japanese display labels, image URL helpers, upgrade-cost helpers, and an in-memory build API.

## Features

- Agents through ZZZ Version 3.0
- W-Engines, drive discs, materials, and latest upstream item catalog data
- Japanese labels via `lang=ja`
- Hakush/Wiki icon URLs with generated SVG fallbacks
- Agent level-cost helper
- In-memory build save/fetch API
- Swagger UI at `/docs`

## Localization

Add `lang=ja` to supported endpoints to receive Japanese display labels under `localized`.

Example:

```txt
GET /api/agents/ellen?lang=ja
```

Base fields remain stable English/API values. Japanese labels are additive so existing clients do not break.

## Images

Catalog responses include `images.icon`, `images.generatedIcon`, and `images.card`.

Example:

```json
{
  "id": "ellen",
  "name": "Ellen Joe",
  "images": {
    "icon": "https://fastcdn.hoyoverse.com/content-v2/nap/113671/59a98a82a886f49c3b1b89bdc77cbb36_21606237077173472.png",
    "generatedIcon": "/api/assets/agents/ellen/icon.svg",
    "card": "https://fastcdn.hoyoverse.com/content-v2/nap/113671/59a98a82a886f49c3b1b89bdc77cbb36_21606237077173472.png",
    "source": "hoyoverse-official"
  }
}
```

Agent `images.icon` and `images.card` use mapped HoYoverse official site assets when available. If no asset is mapped for a non-agent catalog item, it falls back to the Zenless Zone Zero Wiki file redirect. `images.generatedIcon` remains available as a local fallback generated by this API.

For agents and other curated catalog entries, `/api/assets/:kind/:id/icon.svg` and `/card.svg` are generated SVG fallbacks. Use `/api/assets/:kind/:id/official-icon` and `/official-card` when you want the mapped official/source image URL.

All icon URLs can also be fetched as a manifest:

```txt
GET /api/icons?lang=ja
GET /api/icons/w-engines
GET /api/icons/items?q=denny
```

Latest upstream items from `zzz.nanoka.cc` include `images.generatedIcon`, `images.generatedCard`, and the upstream `images.sourcePath` from the game data. The generated SVG endpoints are local fallbacks for clients that need a stable image URL.

## Notes

This project does not use HoYoLAB credentials or private account data.

Agent data is seeded through Version 3.0 as of 2026-06-29. `/api/agents` returns released agents by default. Use `includeUpcoming=true` or `/api/agents/upcoming` to include announced Version 3.0 upcoming agents.

W-Engines, drive discs, and materials are implemented as catalog records. `/api/items` is powered by the latest public `zzz.nanoka.cc` item dataset and supports search, rank filtering, class filtering, and pagination. Some long effect text can be refined later without changing stable IDs.

## Latest Upstream Data

The API can also inspect the latest public data version from `zzz.nanoka.cc`.

```txt
GET /api/source/latest
GET /api/source/latest/summary
GET /api/source/latest/diff
GET /api/source/latest/character
GET /api/source/latest/character?preview=true&limit=5
GET /api/source/latest/weapon
GET /api/source/latest/equipment
GET /api/source/latest/item
```

These endpoints return upstream data separately from the curated local catalog. Use `/summary` for record counts, `/diff` for local-vs-upstream count comparisons, and `preview=true` to inspect a small slice without downloading the full dataset.
